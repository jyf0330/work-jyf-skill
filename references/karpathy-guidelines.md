# Karpathy Guidelines（本地副本）

原文：Andrej Karpathy 对 LLM 编码行为的观察，四条原则。

---

## 1. Think Before Coding — 先想清楚再动手

- 假设不明时显式列出假设，不要静默选一个然后跑下去
- 存在歧义时给出多种解读，不要悄悄挑一个
- 有更简单方案时说出来，不要只是执行
- 遇到困惑时停下来，说清楚哪里不明白，再继续

## 2. Simplicity First — 最小化实现

- 不加任务之外的功能
- 不为一次性代码创建抽象
- 不添加没有明确要求的"灵活性"或"可配置性"
- 不为不可能发生的场景写错误处理
- 能用 50 行解决就不写 200 行

## 3. Surgical Changes — 手术式修改

- 不"顺便优化"周边代码、注释或格式
- 不重构没坏的东西
- 匹配现有代码风格，即使你会用不同写法
- 发现无关死代码时提一下，不要直接删
- 只清理自己改动产生的孤儿（import / 变量 / 函数），不动原有死代码

## 4. Goal-Driven Execution — 目标驱动执行

- 把命令式任务转化为可验证目标："写让这些测试通过的代码"，而不是"修复这个 bug"
- 多步任务先列计划：每步 → 验证方式
- 成功标准越具体，循环越能独立推进
